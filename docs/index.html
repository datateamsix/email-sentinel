3 Sample Use Cases

Job Alerts

Stop refreshing LinkedIn every 10 minutes. Get instant alerts for interview invites, recruiter messages, and application updates‚Äîeven when buried in promotions.

email-sentinel filter add \
  --name "Job Opportunities" \
  --from "linkedin.com,greenhouse.io" \
  --subject "interview,opportunity" \
  --scope "inbox+promotions" \
  --labels "career,urgent"

OTP Codes

Stop switching between email and your login screen. Email Sentinel auto-extracts 2FA codes and copies them to your clipboard the second they arrive.

email-sentinel filter add \
  --name "Verification Codes" \
  --from "noreply@github.com,google.com" \
  --subject "verification,code,2fa" \
  --labels "otp,security"

VIP Alerts

Your boss emails at 11 PM. You see it at 11:01. Never miss time-sensitive messages from executives, clients, or anyone who signs your paychecks.

email-sentinel filter add \
  --name "Boss & Execs" \
  --from "ceo@company.com,boss@company.com" \
  --scope inbox \
  --labels "work,vip,urgent"

Create A Smart Email Warehousing System

Monitor unlimited email accounts through one central Gmail inbox

What Is Email Warehousing?

Instead of connecting multiple email accounts to Email Sentinel, you forward important emails from all your accounts (work, personal, client) into one central Gmail inbox. Email Sentinel monitors that single inbox.

Why Use It?

‚úÖ One OAuth authentication, unlimited accounts
‚úÖ Unified monitoring of all important messages
‚úÖ Original sender info preserved in forwards
‚úÖ Multi-stage filtering (email client ‚Üí Gmail ‚Üí Sentinel)
‚úÖ No complex multi-account OAuth setup

Quick Setup

  1. Create a new Gmail account (e.g., my-alerts-hub@gmail.com)
  2. Set up email forwarding from your other accounts (Gmail, Outlook, iCloud, etc.)
  3. Connect Email Sentinel to your central inbox: email-sentinel init
  4. Add filters to catch forwarded emails from specific sources

Quick Installation

# Using Homebrew (recommended)
brew tap datateamsix/tap
brew install email-sentinel

# Initialize and authenticate
email-sentinel init

# Add your first filter
email-sentinel filter add

# Start monitoring with system tray
email-sentinel start --tray --ai-summary
# Using Scoop (recommended)
scoop bucket add datateamsix https://github.com/datateamsix/scoop-bucket
scoop install email-sentinel

# Initialize and authenticate
email-sentinel init

# Add your first filter
email-sentinel filter add

# Start monitoring with system tray
email-sentinel start --tray --ai-summary
# Debian/Ubuntu
wget https://github.com/datateamsix/email-sentinel/releases/latest/download/email-sentinel_*_amd64.deb
sudo dpkg -i email-sentinel_*_amd64.deb

# RHEL/Fedora/CentOS
wget https://github.com/datateamsix/email-sentinel/releases/latest/download/email-sentinel_*_x86_64.rpm
sudo rpm -i email-sentinel_*_x86_64.rpm

# Initialize and authenticate
email-sentinel init

# Start monitoring
email-sentinel start --tray
# Pull the latest image
docker pull datateamsix/email-sentinel:latest

# Run with mounted config directory
docker run -v ~/.email-sentinel:/root/.email-sentinel \
  datateamsix/email-sentinel:latest start --daemon

# Or use docker-compose
curl -O https://raw.githubusercontent.com/datateamsix/email-sentinel/main/docker-compose.yml
docker-compose up -d
